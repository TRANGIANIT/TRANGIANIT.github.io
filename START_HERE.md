## 💡 Key Features

### ✨ Smart Algorithm (SM-2)
```
EF = Ease Factor (determines how long between reviews)
New EF = Old EF + (0.1 - (5 - quality) × 0.08)

Quality Levels:
❌ Again  (0) → EF down → Review in 1 day
😐 Hard   (1) → EF down → Review in 3 days
😊 Good   (3) → EF same → Review in 7 days
😃 Easy   (4) → EF up   → Review in 21 days

Result: Personalized learning intervals!
```

## 🔍 Understanding the System

### How It Works (Simple Version)
```
You see a flashcard
    ↓
You try to remember
    ↓
You check the answer by clicking
    ↓
You rate difficulty: Easy/Good/Hard/Again
    ↓
System calculates next review date
    ↓
Tomorrow: Only review cards due today
    ↓
Over time: Less frequent reviews for mastered cards
    ↓
Result: Learn more, study less! 🎉
```

### How It Works (Technical Version)
```
User clicks quality button
    ↓
recordResponse(quality) function runs
    ↓
SM-2 algorithm calculates new Ease Factor
    ↓
Interval calculated: 1/3/7/21 days
    ↓
Due date = Today + Interval
    ↓
Save to Firebase: card_progress + review_history
    ↓
Dashboard queries: "What cards are due today?"
    ↓
Dashboard updates with new stats
    ↓
Next session: Only shows cards due today
```
